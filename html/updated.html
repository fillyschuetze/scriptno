Happy (belated) (Lunar) new year to everyone! I'd like to sincerely thank Steve Gibson (and Leo Laporte) for featuring and praising ScriptNo on Security Now #339, and to all the generous individuals who have supported me. I would like to personally email all of you, but I don't have the luxury of time.

Some of you asked why the donation button isn't more obvious (instead of a green heart); the main reason is I don't want to obstruct your browsing experience with "PAY ME NOW"-esque messages. A donation is a donation: voluntary and out of generosity (not coerced).

Without further ado, here are the additions/changes to ScriptNo in this release:
  • v1.0.6.1: addresses issues 64, 66, 72, and 73; able to set custom referrer values now; added FAQs link to tab popup; minor tweaks to Options page and tab popup; please check out the updated FAQs page to see if it answers any questions you may have :) (especially playing nicely with other extensions)
  • Less memory usage (no exact quantity, but it seems to be a significant decrease)
  • Truly blocks inline scripts
  • New features:
    • Block Unwanted Cookies - block cookies from being set by domains in the unwanted content domain list (Default: Enabled)
    • User-Agent Spoof - spoof your browser and operating system (Default: Disabled)
    • Referrer Spoof - spoofs the referrer for all requested resources (options: Same Document, Same Domain, Off) (Default: Same Domain)
    • Block Cross-Domain XML - block cross-domain XML HTTP Requests (Default: Enabled)
  • Shows why a domain has been blocked in the tab widget (e.g. "Unwanted" or "Antisocial")
  • Fixed Issue 42, Issue 46, Issue 49, Issue 59, Issue 60
  • Updated unwanted content domains list (domains compiled and optimized from MVPS HOSTS, hpHOSTS (ad / tracking servers), Peter Lowe's HOSTS Project, MalwareDomainList.com, and DNS-BH â€“ Malware Domain Blocklist)
  • Some code and interface polishing
  • Made "Disable and Remove <NOSCRIPT>" disabled by default
My goals for the future:
  • Language translations
  • Cross-computer settings syncing
  • And, of course, fixing as many reported issues as possible
Again, thank you, and enjoy.
